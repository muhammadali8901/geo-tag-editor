📢 Promotional Ads Component - Implementation Guide

Complete, production-ready code for integrating promotional ads into the Geo Tag Editor website.

Files Required:

  • /css/promotional-ads.css - Component styles (10KB)
  • /js/promotional-ads.js - Component functionality (8KB)

🚀 1. Setup Instructions

Step 1: Add Stylesheet Link

Add this line in the <head> section of your pages:

<link rel="stylesheet" href="/css/promotional-ads.css">

Step 2: Add JavaScript Module

Add this line before </body> closing tag:

<script src="/js/promotional-ads.js" defer></script>

Step 3: Copy Component HTML (see examples below)

Choose between card ad or sidebar ad based on placement.

✅ Pro Tip: Add these links to your main pages:
  • index.html (Homepage)
  • geo-tag-editor/index.html (Tool page)
  • geotag-photos-online/index.html (Tool page)
  • how-to-remove-location-from-photos/index.html (Removal tool)

⚙️ 7. JavaScript API & Methods

Available Methods (window.GeoTagAds):

Toggle Ads On/Off:

// Hide all ads
GeoTagAds.toggleAds(false);

// Show all ads (default)
GeoTagAds.toggleAds(true);

Update Ad Content Dynamically:

// Update specific ad card
GeoTagAds.updateAdContent('.ad-card-container', {
  title: 'New Title',
  subtitle: 'New subtitle text',
  buttonText: 'New Button Text',
  buttonUrl: 'https://new-url.com'
});

Configuration (in promotional-ads.js):

GeoTagAds.config = {
  sidebarSelector: '.ad-sidebar-container',
  iframeSelector: '.ad-iframe-wrapper iframe',
  sidebarSticky: true,              // true = sticky, false = fixed
  stickyOffset: 100,                // pixels from top
  lazyLoadThreshold: 100            // pixels before visible area
};

If Google Analytics is loaded, clicks are automatically tracked:

⚡ 8. Performance & Optimization Tips

1. Lazy Load iframes:

<!-- Use data-src + loading="lazy" -->
<iframe 
  data-src="https://your-listing-website.com"
  loading="lazy"
  title="Preview">
</iframe>

2. Component Sizes:

  • CSS: 10KB (minified: ~3KB)
  • JS: 8KB (minified: ~2.5KB)
  • Total impact: ~5-6KB with minification + gzip

3. Lighthouse Optimization:

  • ✅ No render-blocking scripts (JS is deferred)
  • ✅ No cumulative layout shift (CLS)
  • ✅ Iframes lazy-loaded by default
  • ✅ Minimal repaints/reflows
  • ✅ CSS is non-critical (can be deferred)

4. Advanced: Async CSS Loading

  href="/css/promotional-ads.css" 
  as="style" 
  onload="this.onload=null;this.rel='stylesheet'">
<noscript><link rel="stylesheet" href="/css/promotional-ads.css"></noscript>
✅ SEO Friendly:
  • Proper semantic HTML (role, aria-label attributes)
  • Does not interfere with content indexing
  • Print-friendly (ads hidden on print)

🔧 9. Troubleshooting & FAQs

Q: Ads not appearing on mobile?

A: This is expected. Use CSS media queries to customize. Sidebar auto-hides below 1024px. Card ads remain full-width on mobile.

Q: Iframe not loading?

A: Check:

  • URL is correct in data-src attribute
  • Cross-origin restrictions (add sandbox attribute if needed)
  • Your listing site allows embedding (check X-Frame-Options header)
  • Browser console for CORS errors

Q: Sidebar overlapping content?

A: Adjust sidebar width or position:

.ad-sidebar-container {
  width: 100px;        /* Wider sidebar */
  right: 40px;         /* More margin from edge */
  z-index: 90;         /* Lower than critical UI */
}

Q: Want to hide ads programmatically?

A: Use the JavaScript API:

// Hide ads after 30 seconds
setTimeout(() => GeoTagAds.toggleAds(false), 30000);

Q: Analytics not tracking?

A: Ensure Google Analytics (GA4) is loaded before ads.js. Check browser console for debug messages.

Q: Sidebar position wrong on mobile?

A: Sidebar is hidden by default on mobile (breakpoint: 1024px). To show on mobile:

/* Override default behavior */
@media (max-width: 1024px) {
  .ad-sidebar-container {
    display: flex !important;      /* Force visible */
    position: static;              /* Switch to static positioning */
    width: 100%;
    margin-top: 20px;
  }
}
