Why a Photo's GPS Is Often Wrong

A camera's GPS reading is a best guess, not a measurement. The accuracy depends on how good the satellite fix was at the moment the shutter fired, and that's affected by a long list of things you have no control over at the time. The most common reasons coordinates end up off:

  • Indoor and urban canyon shots. Walls and tall buildings block satellite signals; the phone falls back to Wi-Fi or cell-tower triangulation, which can be off by 50–500 metres.
  • Cold-start GPS. If the phone hasn't had a fix recently, the first photo of the session may use the last known location instead of the current one.
  • Assisted GPS errors. Some phones write a coordinate before they've actually locked on, leaving early shots from a trip tagged at home or at the airport.
  • Time-zone and clock drift. If the camera's clock is wrong, a tracker-based geotagging workflow (matching photos to a GPX log) snaps to the wrong point on the route.
  • Imported tags from other software. Some apps round coordinates aggressively or apply them at the album level, leaving every photo at a single approximate point.

How the Editor Handles Existing GPS

When you drop a JPEG into the tool, it reads the file's EXIF segment and looks for the GPS IFD. If coordinates are present, the latitude and longitude inputs are pre-filled with the existing values and the map marker is placed at that point. From there you have three options:

  • Drag the marker to the correct location, then click Apply. The new coordinates overwrite the old GPS values; the rest of the EXIF (camera, lens, timestamp) is left intact.
  • Type new latitude/longitude directly — useful if you have the precise coordinates from another source (a survey, a Google Maps pin, a GPX waypoint).
  • Remove the GPS block entirely with the Remove Geotag button if the original is wrong but you don't have a correct value to replace it with.

The image pixels are not touched. The output JPEG has the same resolution and visual quality as the input — only the GPS bytes in the metadata are rewritten.
